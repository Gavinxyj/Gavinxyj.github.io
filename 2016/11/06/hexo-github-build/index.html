hexo+github个人博客搭建(一)

2016-11-06

版权申明:本文为作者原创文章，可以随便转载，但必须在明确位置标明出处！！！

　　最近一直在忙着搭建自己的个人博客，今天总算是完完整整的把个人博客搭建好了，为什么要搭建个人博客呢，原因是现在其它运营商的博客广告太多，我选择hexo来搭建自己的博客，其原因是界面干净利落，阅读体验好。如果你也想搭建自己的个人博客，那么下面会一步一步教你如何搭建自己的个人博客。
　　我们使用hexo + github来搭建自己的个人博客，github我相信每个开发人员对它都不陌生，github目前算是全球最大的开源项目托管平台，像google，百度，阿里都有其开源项目托管在上面，当然github上也汇聚了全球最牛X的大神，github的宗旨是不要重复造轮子，所以在你日常的开发中需要用到什么库先到github上搜索一下也许就有你需要的东西，你可以阅读源码来学习大神们的编程思想，当然等你熟悉了其源码你也可以参与到源码的编写中。开源和分享使人进步。

okay，现在开始搭建我们的个人博客：

  1. 首先在开始搭建博客之前先将我们博客所需要的依赖环境搭建好，我们将我们的博客部署到Ubuntu中，还没有linux系统的可以用虚拟机创建一个linux系统，我自己也是在虚拟机中部署的。
  2. 在命令行终端执行sudo apt-get install nodejs 回车，如果出现让你插入光驱的提示，cd到/etc/apt目录下 vi source.list注释掉deb cdrom这一行，如下图所示：

    然后执行sudo apt-get update命令，待命令执行完毕再执行apt-get install nodejs命令，若不出现意外应该可以安装成功，若出现错误根据错误提示去找解决方法，验证有没有安装成功执行命令node –help如出现下图证明你安装成功了。
  3. 安装git，同样执行命令sudo apt-get install git回车，安装完成执行git –help出现下图证明你安装成功了。
  4. 安装hexo，执行命令sudo npm install -g hexo-cli，安装完成执行命令hexo -help命令出现如下图所示证明安装完成。
  5. 在你的用户目录下执行hexo init blog命令，等待命令执行完成就会在当前目录下生产blog目录，cd blog出现如下图的几个文件。
  6. 到这里我们就把hexo安装完成了，现在我们在当前目录下执行hexo g命令生产静态文件，执行命令hexo s启动本地服务端，端口号默认4000，如下图所示
  7. 打开我们的浏览器输入你虚拟机IP地址加端口号，如我的虚拟机IP地址是192.168.1.128，那么在浏览器上输入192.168.1.128 4000回车出现如下图所示证明你的hexo在本地已经搭建好了。
  8. blog目录下的_config.yml配置文件，大部分配置都在这里，themes主题文件，进入themes文件目录该文件目录下只有一个主题landscape主题，是hexo默认的主题，如果你觉得不好看你可以在hexo.io官网上找你喜欢的主题，这里我自己用的是fexo主题，简单美观，大家可以访问我的博客地址www.gavinxyj.com。主题风格如下图所示：
